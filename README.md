# 技术商业化分析 Skill 使用说明

这个仓库包含一个 Codex skill：`professor-commercialization-research`，用于从学者、课题组或实验室名称出发，自动梳理论文成果、研究方向、商业化机会、国内外商业化公司对标，以及面向技术经理人的学者访谈问题清单。

也可以打开 [HTML 版使用说明](usage.html) 阅读。

## 适用场景

- 想快速了解某位学者近十年的科研方向和关键成果。
- 想判断某些论文或研究方向是否有产品化、公司化或技术转移机会。
- 想形成技术经理人与学者访谈前的研究底稿。
- 想比较国内外商业化公司对标、平台替代和市场空白。

## 唤醒方式与推荐输入

在 Codex 中直接说：

```text
技术商业化分析skill，帮我分析某高校或研究机构的某位学者的成果
```

也可以只说：

```text
技术商业化分析skill
```

如果没有给出机构和学者姓名，skill 会先询问你想研究哪个机构的哪位学者。

如果希望限定研究范围或交付格式，可以补充：

```text
技术商业化分析skill，帮我分析某高校或研究机构的某位学者近十年的成果。
重点关注 AI、数据安全和金融科技方向，输出 Excel 和 HTML。
```

## 工作流程

1. 确认学者身份、机构、姓名变体和公开学术主页。
2. 抓取并清洗论文列表，去重并记录来源。
3. 为每篇论文生成专业摘要、通俗解释和可能应用场景。
4. 聚类研究方向，形成时间线和关键节点。
5. 可选：用户选择若干论文编号，补充商业假设，skill 深入判断是否准确。
6. 形成商业化分析，包括客户、痛点、产品形态、客户会自建还是采购的判断和建议。
7. 形成国内外商业化公司对标或空白判断。
8. 生成面向技术经理人的学者访谈问题清单。

第 5 步是唯一建议用户介入的步骤；如果用户不选择具体论文，skill 会继续做整体商业化分析。

## 主要交付物

默认交付物以 Excel 为主，可同时生成可读性强的 HTML 或 Word。

Excel 通常包含：

- `Overview`：总体一览。
- `Paper Browser`：逐篇论文清单、专业摘要、通俗解释和应用场景。
- `Directions`：研究方向聚类。
- `Timeline`：十年研究演化与关键节点。
- `Deep Dives`：用户选择论文的深入商业假设判断。
- `Commercialization`：商业化机会、客户、产品形态和建议。
- `Benchmarks`：国内外商业化公司对标或空白。
- `Interview Questions`：技术经理人与学者访谈的问题清单。
- `Sources`：来源链接和证据记录。

## 输出风格

- 面向技术经理人，尽可能降低理解研究内容时的技术术语壁垒。
- 商业化判断优先回答：谁有痛点、为什么现在需要、谁付费、客户会自建还是购买、还缺什么资源。
- 技术细节主要作为证据和追问，不作为访谈主轴。
- 分析商业化潜力时做对抗性分析，避免美化或过分乐观地给出建议。

## 使用注意

- 现代论文、公司、政策和市场信息会变化，运行时应联网核验最新资料。
- 不绕过付费墙，只下载开放论文或作者公开版本。
- 如果公开摘要或全文不足，逐篇分析会标注较低证据置信度。
- 交付物适合做前期研判和访谈准备，不替代正式法律、财务、知识产权或投资尽调。
